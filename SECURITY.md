# セキュリティポリシー

## 脆弱性の報告

脆弱性の可能性がある問題を見つけた場合は、**公開 Issue を作成せず**に、以下の窓口へご連絡ください。

- 連絡先: `<SECURITY_CONTACT>`
- 代替手段: GitHub の [Private vulnerability reporting](https://docs.github.com/en/code-security/security-advisories/guidance-on-reporting-and-writing-information-about-vulnerabilities/privately-reporting-a-security-vulnerability)（有効化している場合）

可能であれば、次の情報を含めてください。

- 影響範囲（影響するバージョン/コミット、利用条件）
- 再現手順（PoC があれば添付）
- 想定される影響（情報漏えい、権限昇格、DoS 等）
- 発見者情報（クレジット希望の有無、連絡可能な時間帯など）

## 対応フロー（目安）

| 段階 | 目安期間 | 内容 |
|---|---|---|
| 初回応答 | 3 営業日以内 | 受領確認、担当者アサイン |
| 影響評価 | 7 営業日以内 | 重大度判定、再現確認 |
| 修正 | 重大度に応じて調整 | パッチ作成、リグレッションテスト |
| 公表 | 修正提供後 | アドバイザリ / CVE 発行（必要に応じて） |

> 上記はスターターテンプレの目安値です。プロジェクトごとに実情に合わせて調整してください。

## Safe Harbor（善意の研究者保護）

善意に基づき本ポリシーに従って脆弱性を報告・調査する研究者に対して、
本プロジェクトは報復的な法的措置を取らないことを約束します。
ただし、以下の行為は対象外です。

- サービスの可用性を損なう行為（DoS 等）
- 他ユーザーのデータへのアクセス・改ざん
- ソーシャルエンジニアリング、物理的侵入

## PGP / 暗号化

機微な情報を送る場合は、暗号化メールや GitHub の Private vulnerability reporting を利用してください。
PGP 公開鍵が必要なプロジェクトでは、ここに鍵 ID / フィンガープリントを記載してください。
